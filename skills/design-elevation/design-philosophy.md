# Design Philosophy

Principles for making design decisions. These guide the judgment calls that techniques alone cannot answer.

---

## Core Principles

### 1. Intentionality Over Intensity

Every choice should have a reason. Bold maximalism and refined minimalism both work—the key is intentionality, not intensity.

**Ask**: Why this color? Why this font? Why this spacing?
**If answer is "default" or "I don't know"**: Change it.

Intention doesn't mean complexity. A single typeface at one size is intentional if chosen deliberately. Random variety is not intentional even if it took more effort.

### 2. Expertise With Restraint

Professional designers make bold choices but execute them with taste. Amateurs either play it safe (boring) or go wild (chaotic).

**Bold choice, restrained execution**:
- Dramatic color palette, applied sparingly
- Oversized typography, limited instances
- Unusual layout, consistent throughout

**What to avoid**:
- Safe choices with heavy decoration (lipstick on a pig)
- Bold choices applied everywhere (visual noise)

### 3. Context-Appropriate Boldness

The right level of visual energy depends on the purpose:

| Context | Energy Level | Approach |
|---------|--------------|----------|
| Legal document | Low | Swiss, minimal |
| Sales presentation | Medium-High | Memorable, branded |
| Event invitation | High | Distinctive, festive |
| Technical docs | Low-Medium | Clear, functional |
| Marketing page | Medium-High | Engaging, persuasive |
| Dashboard | Low-Medium | Scannable, efficient |

Match the energy. A financial report shouldn't feel like a party invitation. A wedding invite shouldn't feel like a tax form.

### 4. The Hand-Crafted Test

Ask: Would someone think a human designer spent time on this?

**Signs of hand-crafted**:
- Thoughtful typography pairing
- Intentional color relationships
- Details that serve a purpose
- Consistent visual language
- Something memorable

**Signs of template/generated**:
- Default fonts unchanged
- Arbitrary color choices
- Inconsistent styling
- Nothing distinctive
- Could be any company

### 5. One Memorable Thing

Every output should have at least one element that sticks:
- Striking typography treatment
- Unusual but appropriate color
- Distinctive layout approach
- Delightful detail
- Clear visual signature

If someone describes your output to someone else, what would they mention? If nothing, it's too generic.

## Avoiding Anti-Patterns

### Generic Defaults

**Problem**: Using whatever comes out of the box.

**Examples**:
- System fonts (San Francisco, Segoe, Arial)
- Bootstrap colors unchanged
- Default chart styles
- Stock template layouts

**Fix**: Make at least one deliberate change in each category (typography, color, layout). Even small changes signal intention.

### Template Aesthetics

**Problem**: Output that looks obviously templated.

**Signs**:
- Placeholder-looking imagery
- Generic icons from icon packs
- Standard layouts everyone uses
- Nothing connecting to actual brand/context

**Fix**: Customize for context. Use real content. Add one distinctive element.

### Over-Decoration

**Problem**: Adding visual elements without purpose.

**Examples**:
- Gradients for gradient's sake
- Shadows on everything
- Decorative shapes that obscure content
- Animation that distracts

**Fix**: Ask "what does this add?" for every decorative element. If the answer is just "visual interest," reconsider.

### Timid Execution

**Problem**: Half-committing to design decisions.

**Examples**:
- Colors at 70% saturation when they should be bold
- Sizes that are "safe" middle ground
- Contrast that's present but not confident
- Layouts that are "almost" interesting

**Fix**: Commit fully. If a choice is right, execute it with confidence. If it seems too bold, maybe the choice is wrong—pick something else entirely.

## Making Judgment Calls

### When To Follow Rules vs. Break Them

**Follow rules when**:
- Building trust is paramount
- Content is primary (documentation, reports)
- Consistency matters (design systems, brand)
- Audience expects convention

**Break rules when**:
- Standing out is the goal
- Creativity is valued
- One-off piece (not a system)
- Audience appreciates surprise

### When Less Is More

- Dense information (dashboards, data tables)
- Professional contexts (legal, financial, medical)
- Content-heavy formats (articles, documentation)
- When typography does the work

### When More Is More

- Celebration/events
- Creative/artistic contexts
- Marketing/persuasion
- Making a statement

## The Ultimate Questions

Before delivering anything:

1. **Is this the best I can do?** Not "is this acceptable"—is this excellent?

2. **Would I be proud to show this?** Would I put this in a portfolio?

3. **Does this serve the user?** Beyond looking good, does it work?

4. **Is there soul here?** Is there evidence of care and consideration?

5. **What would I change if I had more time?** Then change it now.

---

## Summary

Good design is:
- **Intentional**: Every choice has a reason
- **Appropriate**: Matches context and audience
- **Distinctive**: Has memorable quality
- **Restrained**: Bold where it matters, simple elsewhere
- **Crafted**: Shows evidence of care

Good design is not:
- Decoration for decoration's sake
- Following defaults
- Playing it safe
- Trying everything
- Imitating without understanding
